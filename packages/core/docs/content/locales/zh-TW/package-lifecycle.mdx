# 套件生命週期

可重用套件可以發布具版本的靜態 `agent-native.package.json`。CLI 檢查時只會
讀取此 JSON，不會執行套件程式碼。

```bash
agent-native package inspect @agent-native/scheduling --json
agent-native package add @agent-native/scheduling --apply
agent-native package eject @agent-native/scheduling --apply
```

新增與退出預設只會預覽。只有指定 `--apply` 才會寫入檔案並執行偵測到的套件
管理器。CLI 會拒絕衝突與未知配置；安裝失敗時會還原檔案與鎖定檔。

## 退出單一功能

請依序自訂：設定公開 API、組合公開元件、退出最小單元；若多個應用程式都
需要相同變更，再提出共用擴充介面。

```bash
agent-native eject --list
agent-native eject inspect <unit>
agent-native eject <unit> --app <app>
agent-native eject <unit> --app <app> --apply
agent-native eject diff <unit> --app <app>
agent-native eject restore <unit> --app <app> --apply
```

這些命令預設只會預覽。套用後，會將套件版本、資訊清單摘要、檔案雜湊與
import 重寫記錄到可提交的 `agent-native.ejections.json`。Restore 只會在檔案
與 import 維持不變時執行；否則會顯示差異並拒絕刪除本機修改。

第一方資訊清單涵蓋所有公開 Toolkit 進入點及第一方整合目錄。缺少第一方配方
屬於框架涵蓋錯誤；未知的第三方套件會收到 add 形式的藍圖。受保護的 Core
契約不會被複製，CLI 會引導至公開設定、轉接器或擴充介面。

## 遷移

已發佈套件包含記錄移動路徑與符號的 `migration-manifest.json`。`npx @agent-native/core@latest upgrade --codemods` 會預覽差異，檢閱後用 `--yes` 套用。CI 請執行 `agent-native doctor --only migration-manifest`；此檢查沒有 opt-out。

`planned` 項目可在目標之前發佈；它們只會顯示為非阻塞警告，codemod 不會重寫。active 移動只有在目標 export 已安裝時才會重寫。
