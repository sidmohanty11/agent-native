# Primeiros passos

Agent-Native é para criar agentic applications: o AI agent e a UI compartilham as mesmas [actions](/docs/actions), dados SQL e application state. O caminho principal começa pelo Chat para que usuários possam falar com o agent imediatamente, e depois adiciona as superfícies de app que o fluxo merece.

O fluxo útil:

1. Criar uma Chat app.
2. Adicionar uma action.
3. Renderizar o resultado da action inline no chat.
4. Persistir dados em SQL.
5. Adicionar uma página que o agent pode abrir quando a inspeção visual fizer mais sentido.

## 1. Crie uma Chat app {#create-your-app}

Você precisa de [Node.js 22+](https://nodejs.org) e [pnpm](https://pnpm.io).

```bash
npx @agent-native/core@latest create my-app --template chat
cd my-app
pnpm install
pnpm dev
```

O template inclui durable chat threads, auth, live sync, diretório `actions/`, actions padrão `view-screen` e `navigate`, e uma app React pronta para estender.

## 2. Adicione uma action {#add-an-action}

Uma action é uma operação tipada que o agent e a UI chamam juntos. Para este fluxo, crie `actions/analyze-responses.ts`: ela analisa respostas de formulários e retorna Response insights.

```ts
export default defineAction({
  description: "Analyze recent form responses and summarize the trend.",
  outputSchema: dataInsightsWidgetResultSchema,
  chatUI: {
    renderer: ACTION_CHAT_UI_DATA_INSIGHTS_RENDERER,
    title: "Response insights",
  },
  readOnly: true,
  run: async () =>
    createDataInsightsWidgetResult({
      title: "Response insights",
      display: {
        title: "42 responses",
        description: "Completion rose this week.",
        primaryAction: {
          label: "Abrir insights de respostas",
          href: "/response-insights",
        },
      },
      chartSeries: {
        type: "bar",
        xKey: "day",
        series: [{ key: "responses", label: "Responses" }],
        data: Array.from({ length: 10 }, (_, day) => ({
          day: day + 1,
          responses: Math.round(Math.random() * 10) + 10,
        })),
      },
    }),
});
```

Execute diretamente:

```bash
pnpm action analyze-responses --formId demo
```

Ou peça ao agent no navegador:

> Analise as respostas recentes do formulário de demonstração.

## 3. Renderize o resultado no chat {#render-inline}

Como a action declara `outputSchema` e `chatUI.renderer`, o transcript não transforma dados estruturados em texto corrido. Ele renderiza um componente React nativo: resumo, gráfico, tabela e link para a página detalhada. Veja [Native Chat UI](/docs/native-chat-ui).

Para criar seu próprio gráfico em vez de usar os data widgets integrados, dê à action um id de renderer específico do produto e retorne seu próprio formato JSON validado:

```ts
// actions/analyze-responses.ts
import { defineAction } from "@agent-native/core/action";
import { z } from "zod";

const responseChartRenderer = "responses.response-chart";

const responseChartResultSchema = z.object({
  title: z.string(),
  points: z.array(
    z.object({
      day: z.string(),
      responses: z.number(),
    }),
  ),
});

export default defineAction({
  description: "Analyze recent form responses and render a custom chart.",
  schema: z.object({
    formId: z.string().default("demo"),
  }),
  outputSchema: responseChartResultSchema,
  chatUI: {
    renderer: responseChartRenderer,
    title: "Response chart",
  },
  readOnly: true,
  run: async ({ formId }) => ({
    title: `Responses for ${formId}`,
    points: [
      { day: "Mon", responses: 12 },
      { day: "Tue", responses: 18 },
      { day: "Wed", responses: 24 },
    ],
  }),
});
```

Depois registre um componente React para esse id exato no startup da app, por exemplo importando `app/chat-renderers.tsx` uma vez a partir de `app/root.tsx`:

```tsx
// app/chat-renderers.tsx
import {
  registerActionChatRenderer,
  type ToolRendererProps,
} from "@agent-native/core/client/chat";

type ResponseChartResult = {
  title: string;
  points: Array<{ day: string; responses: number }>;
};

function ResponseChart({ context }: ToolRendererProps) {
  const result = context.resultJson as ResponseChartResult;
  const max = Math.max(...result.points.map((point) => point.responses), 1);

  return (
    <section className="rounded-lg border bg-card p-4">
      <h3 className="text-sm font-medium">{result.title}</h3>
      <div className="mt-4 flex h-32 items-end gap-2">
        {result.points.map((point) => (
          <div
            key={point.day}
            className="flex flex-1 flex-col items-center gap-2"
          >
            <div
              className="w-full rounded-t bg-primary"
              style={{ height: `${(point.responses / max) * 100}%` }}
            />
            <span className="text-xs text-muted-foreground">{point.day}</span>
          </div>
        ))}
      </div>
    </section>
  );
}

registerActionChatRenderer({
  id: "responses.response-chart",
  renderer: "responses.response-chart",
  Component: ResponseChart,
});
```

## 4. Persista dados em SQL {#persist-data}

Quando o resultado precisa ser revisitado, comparado ou atualizado, salve em SQL e mantenha leitura/escrita atrás de actions:

- `create-response-insight` escreve um novo insight.
- `list-response-insights` lê dados para a página.
- `update-response-insight` atualiza um registro.
- `analyze-responses` continua retornando o widget nativo do chat.

Não crie uma REST route duplicada só para o navegador; UI e agent devem compartilhar actions.

## 5. Adicione uma página que o agent pode abrir {#add-a-page}

Adicione a route `/response-insights` com `useActionQuery("list-response-insights")` e `useActionMutation("create-response-insight")`. A página é uma projeção do SQL state; os dados continuam sendo escritos pelas mesmas actions.

Estenda `view-screen` e `navigate` para que o agent saiba a route atual e possa abrir `/response-insights`. Resultados também podem incluir `href: "/response-insights"` para mostrar um botão "Abrir insights de respostas".

Quando o chat em página inteira abrir uma página da app, use `AgentChatSurface`, `AgentSidebar`, `useAgentChatHomeHandoff`, `useAgentChatHomeHandoffLinks` e `chatViewTransition` de [Agent Surfaces](/docs/agent-surfaces#rich-chat) para mover o chat para a lateral sem perder o thread.

## Estrutura do projeto {#project-structure}

```text
my-app/
  actions/         # Operações acionáveis pelo agente e pela interface
  app/             # Rotas, páginas e superfícies de chat em React
  server/          # Servidor Nitro e esquema SQL
  AGENTS.md        # Instruções sempre ativas para o agente do app
  .agents/         # Skills que o agente carrega quando relevante
  data/app.db      # Estado SQLite local quando DATABASE_URL não está definido
```

## Quer um ponto de partida completo para analytics? {#analytics-starting-point}

O exemplo de Response insights acima é pequeno de propósito para mostrar as peças do framework. Se você estiver criando um produto real de analytics, comece com [Analytics](/docs/template-analytics). É o ponto de partida robusto: conecte seus providers, use os dashboards e agent actions existentes e personalize a partir daí.
