# 패키지 수명 주기

재사용 패키지는 버전이 지정된 정적 `agent-native.package.json`을 게시할 수
있습니다. CLI는 검사 중 이 JSON만 읽고 패키지 코드를 실행하지 않습니다.

```bash
agent-native package inspect @agent-native/scheduling --json
agent-native package add @agent-native/scheduling --apply
agent-native package eject @agent-native/scheduling --apply
```

추가와 꺼내기는 기본적으로 미리 보기입니다. `--apply`를 지정해야 파일을 쓰고
패키지 관리자를 실행합니다. 충돌과 알 수 없는 레이아웃은 거부하며 설치가
실패하면 파일과 잠금 파일을 복원합니다.

## 기능 하나 꺼내기

사용자 지정은 공개 API 구성, 공개 프리미티브 조합, 가장 작은 단위 꺼내기,
여러 앱에 같은 변경이 필요할 때 공유 seam 제안 순서로 진행합니다.

```bash
agent-native eject --list
agent-native eject inspect <unit>
agent-native eject <unit> --app <app>
agent-native eject <unit> --app <app> --apply
agent-native eject diff <unit> --app <app>
agent-native eject restore <unit> --app <app> --apply
```

명령은 기본적으로 미리 보기입니다. 적용하면 패키지 버전, 매니페스트 digest,
파일 hash, import 재작성 정보를 커밋 가능한 `agent-native.ejections.json`에
기록합니다. Restore는 파일과 import가 그대로일 때만 실행되며, 변경되었다면
diff를 보여 주고 로컬 수정을 삭제하지 않습니다.

First-party 매니페스트는 모든 공개 Toolkit 진입점과 통합 카탈로그를
포함합니다. 누락된 first-party recipe는 framework coverage 오류이며, 알 수 없는
third-party 패키지에는 add 형식 blueprint를 제공합니다. 보호된 Core contract는
복사하지 않고 공개 configuration, adapter 또는 extension seam을 안내합니다.

## 마이그레이션

게시된 패키지에는 이동한 경로와 심볼용 `migration-manifest.json`이 들어 있습니다. `npx @agent-native/core@latest upgrade --codemods`로 diff를 확인한 뒤 `--yes`로 적용하세요. CI에서는 `agent-native doctor --only migration-manifest`를 실행하며 opt-out은 없습니다.

`planned` 항목은 대상보다 먼저 배포할 수 있으며, 차단하지 않는 경고로 표시되고 codemod가 다시 쓰지 않습니다. active 이동은 대상 export가 설치된 경우에만 다시 씁니다.
