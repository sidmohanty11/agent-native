# Primeros pasos

Agent-Native sirve para crear agentic applications: el AI agent y la UI comparten las mismas [actions](/docs/actions), datos SQL y application state. El camino principal empieza con Chat para que los usuarios puedan hablar con el agent de inmediato, y después añade las superficies de producto que el flujo necesita.

El recorrido útil es:

1. Crear una Chat app.
2. Añadir una action.
3. Renderizar el resultado de la action dentro del chat.
4. Persistir datos en SQL.
5. Añadir una página que el agent pueda abrir cuando conviene inspeccionar el resultado visualmente.

## 1. Crea una Chat app {#create-your-app}

Necesitas [Node.js 22+](https://nodejs.org) y [pnpm](https://pnpm.io).

```bash
npx @agent-native/core@latest create my-app --template chat
cd my-app
pnpm install
pnpm dev
```

El template incluye durable chat threads, auth, live sync, un directorio `actions/`, las actions estándar `view-screen` y `navigate`, y una app React lista para extender.

## 2. Añade una action {#add-an-action}

Una action es una operación tipada que comparten el agent y la UI. Para este flujo, crea `actions/analyze-responses.ts`: analiza respuestas de formularios y devuelve Response insights.

```ts
export default defineAction({
  description: "Analyze recent form responses and summarize the trend.",
  outputSchema: dataInsightsWidgetResultSchema,
  chatUI: {
    renderer: ACTION_CHAT_UI_DATA_INSIGHTS_RENDERER,
    title: "Response insights",
  },
  readOnly: true,
  run: async () =>
    createDataInsightsWidgetResult({
      title: "Response insights",
      display: {
        title: "42 responses",
        description: "Completion rose this week.",
        primaryAction: {
          label: "Abrir insights de respuestas",
          href: "/response-insights",
        },
      },
      chartSeries: {
        type: "bar",
        xKey: "day",
        series: [{ key: "responses", label: "Responses" }],
        data: Array.from({ length: 10 }, (_, day) => ({
          day: day + 1,
          responses: Math.round(Math.random() * 10) + 10,
        })),
      },
    }),
});
```

Pruébala directamente:

```bash
pnpm action analyze-responses --formId demo
```

O pídeselo al agent en el navegador:

> Analiza las respuestas recientes del formulario de demo.

## 3. Renderiza el resultado en chat {#render-inline}

Como la action declara `outputSchema` y `chatUI.renderer`, el transcript no convierte los datos en texto plano. Renderiza un componente React nativo: resumen, gráfico, tabla y un enlace hacia la página más profunda. Consulta [Native Chat UI](/docs/native-chat-ui).

Si quieres crear tu propio gráfico en vez de usar los data widgets integrados, dale a la action un id de renderer específico del producto y devuelve tu propia forma JSON validada:

```ts
// actions/analyze-responses.ts
import { defineAction } from "@agent-native/core/action";
import { z } from "zod";

const responseChartRenderer = "responses.response-chart";

const responseChartResultSchema = z.object({
  title: z.string(),
  points: z.array(
    z.object({
      day: z.string(),
      responses: z.number(),
    }),
  ),
});

export default defineAction({
  description: "Analyze recent form responses and render a custom chart.",
  schema: z.object({
    formId: z.string().default("demo"),
  }),
  outputSchema: responseChartResultSchema,
  chatUI: {
    renderer: responseChartRenderer,
    title: "Response chart",
  },
  readOnly: true,
  run: async ({ formId }) => ({
    title: `Responses for ${formId}`,
    points: [
      { day: "Mon", responses: 12 },
      { day: "Tue", responses: 18 },
      { day: "Wed", responses: 24 },
    ],
  }),
});
```

Después registra un componente React para ese id exacto desde el arranque de la app, por ejemplo importando `app/chat-renderers.tsx` una vez desde `app/root.tsx`:

```tsx
// app/chat-renderers.tsx
import {
  registerActionChatRenderer,
  type ToolRendererProps,
} from "@agent-native/core/client/chat";

type ResponseChartResult = {
  title: string;
  points: Array<{ day: string; responses: number }>;
};

function ResponseChart({ context }: ToolRendererProps) {
  const result = context.resultJson as ResponseChartResult;
  const max = Math.max(...result.points.map((point) => point.responses), 1);

  return (
    <section className="rounded-lg border bg-card p-4">
      <h3 className="text-sm font-medium">{result.title}</h3>
      <div className="mt-4 flex h-32 items-end gap-2">
        {result.points.map((point) => (
          <div
            key={point.day}
            className="flex flex-1 flex-col items-center gap-2"
          >
            <div
              className="w-full rounded-t bg-primary"
              style={{ height: `${(point.responses / max) * 100}%` }}
            />
            <span className="text-xs text-muted-foreground">{point.day}</span>
          </div>
        ))}
      </div>
    </section>
  );
}

registerActionChatRenderer({
  id: "responses.response-chart",
  renderer: "responses.response-chart",
  Component: ResponseChart,
});
```

## 4. Persiste datos en SQL {#persist-data}

Cuando el resultado debe revisarse, compararse o actualizarse, guárdalo en SQL y mantén las lecturas/escrituras detrás de actions:

- `create-response-insight` escribe un insight nuevo.
- `list-response-insights` lee los datos para la página.
- `update-response-insight` actualiza un registro.
- `analyze-responses` sigue devolviendo el widget nativo del chat.

No crees una REST route duplicada solo para el navegador; la UI y el agent deben compartir actions.

## 5. Añade una página que el agent pueda abrir {#add-a-page}

Añade una route `/response-insights` con `useActionQuery("list-response-insights")` y `useActionMutation("create-response-insight")`. La página es una proyección del SQL state; los datos siguen escribiéndose por las mismas actions.

Extiende `view-screen` y `navigate` para que el agent conozca la route actual y pueda abrir `/response-insights`. Los resultados también pueden incluir `href: "/response-insights"` para mostrar un botón "Abrir insights de respuestas".

Cuando el chat de página completa abra una página de app, usa `AgentChatSurface`, `AgentSidebar`, `useAgentChatHomeHandoff`, `useAgentChatHomeHandoffLinks` y `chatViewTransition` desde [Agent Surfaces](/docs/agent-surfaces#rich-chat) para deslizar el chat al panel lateral sin perder el thread.

## Estructura del proyecto {#project-structure}

```text
my-app/
  actions/         # Operaciones invocables por el agente y la interfaz
  app/             # Rutas, páginas y superficies de chat de React
  server/          # Servidor Nitro y esquema SQL
  AGENTS.md        # Instrucciones siempre activas para el agente de la app
  .agents/         # Skills que el agente carga cuando corresponde
  data/app.db      # Estado SQLite local cuando DATABASE_URL no está definido
```

## ¿Quieres un punto de partida completo para analytics? {#analytics-starting-point}

El ejemplo de Response insights anterior es pequeño a propósito para mostrar las piezas del framework. Si vas a crear un producto real de analytics, empieza con [Analytics](/docs/template-analytics). Es el punto de partida robusto: conecta tus providers, usa los dashboards y agent actions existentes, y personaliza desde ahí.
