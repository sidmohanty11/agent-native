# パッケージのライフサイクル

再利用可能なパッケージは、バージョン付きの静的な
`agent-native.package.json` を公開できます。CLI は検査時にこの JSON のみを
読み、パッケージコードを実行しません。

```bash
agent-native package inspect @agent-native/scheduling --json
agent-native package add @agent-native/scheduling --apply
agent-native package eject @agent-native/scheduling --apply
```

追加と排出は既定でドライランです。`--apply` を指定した場合だけ書き込みと
パッケージマネージャーを実行します。競合や未知の配置は拒否し、インストール
失敗時にはファイルとロックファイルを復元します。

## 機能を一つ排出する

カスタマイズは、公開 API の設定、公開プリミティブの合成、最小単位の排出、
複数アプリで同じ変更が必要な場合の共有シーム提案、の順で行います。

```bash
agent-native eject --list
agent-native eject inspect <unit>
agent-native eject <unit> --app <app>
agent-native eject <unit> --app <app> --apply
agent-native eject diff <unit> --app <app>
agent-native eject restore <unit> --app <app> --apply
```

コマンドは既定でドライランです。適用すると、パッケージのバージョン、
マニフェストのダイジェスト、ファイルハッシュ、import の書き換えを
`agent-native.ejections.json` に記録し、リポジトリへコミットできます。
Restore はファイルと import が未変更の場合だけ実行され、それ以外は差分を
表示してローカル変更の削除を拒否します。

First-party マニフェストは、すべての公開 Toolkit エントリポイントと
integration カタログを網羅します。First-party のレシピ欠落は framework の
coverage error です。未知の third-party パッケージには add 形式の blueprint を
出力します。保護された Core contract はコピーせず、公開の設定、adapter、
extension seam を案内します。

## 移行

公開パッケージには移動したパスとシンボル用の `migration-manifest.json` が含まれます。`npx @agent-native/core@latest upgrade --codemods` で差分を確認し、確認後に `--yes` で適用します。CI では `agent-native doctor --only migration-manifest` を実行し、opt-out はできません。

`planned` エントリは移行先より先に公開でき、ブロックしない警告として表示され、codemod は書き換えません。active な移動は、移行先の export がインストール済みの場合にのみ書き換えられます。
